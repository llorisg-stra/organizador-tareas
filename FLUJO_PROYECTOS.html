рҹ“– Las 7 Fases

Fase 1: IDEA вҖ” Nace la iniciativa

Cualquier miembro del equipo propone una iniciativa. Puede ser:

  • Una migraciГіn (email, Passbolt, VPS...)
  • Un proyecto nuevo (web, app, producto...)
  • Una mejora operativa (automatizaciГіn, optimizaciГіn...)

Resultado: Una frase clara que describe quГ© se quiere conseguir.

Ejemplo: "Migrar el dominio de email de @strategialabs.eu a @slabs.studio"

Fase 2: PLAN вҖ” Se documenta y estructura

El agente (Antigravity) crea un plan de implementaciГіn con toda la informaciГіn necesaria para ejecutar el proyecto.

Archivo: proyectos/{nombre-proyecto}/plan.md

proyectos/
в”ңв”Җв”Җ migrar-email-slabs/
в”Ӯ   в”ңв”Җв”Җ plan.md              вҶҗ Plan detallado
в”Ӯ   в”ңв”Җв”Җ task.md              вҶҗ Checklist de seguimiento
в”Ӯ   в””в”Җв”Җ notas/               вҶҗ Capturas, docs auxiliares
в”ңв”Җв”Җ migrar-passbolt/
в”Ӯ   в”ңв”Җв”Җ plan.md
в”Ӯ   в”ңв”Җв”Җ ...

Contenido del plan.md: resumen, datos del proyecto, hitos con fechas, riesgos y criterios de Г©xito.

Fase 3: PLANE вҖ” Se crean los issues

Una vez aprobado el plan, el agente crea un issue por hito en Plane:

Campo Issue Valor
TГӯtulo [Proyecto] Hito X: Nombre del hito
Estado Backlog вҶ’ In Progress вҶ’ Done
Etiqueta SegГәn el tipo (infra, desarrollo...)

Fase 4: HACER вҖ” Se ejecuta el trabajo

Durante la ejecuciГіn, cada sesiГіn actualiza el task.md local con el progreso.

- [x] Hito 1: Agregar dominio en M365
- [/] Hito 2: Configurar DNS Cloudflare
- [ ] Hito 3: Crear alias para el equipo

Fase 5: MARCAR вҖ” Se actualiza el progreso

Cuando un hito se completa: task.md [x] в”Җв”ҖвҶ’ Plane (Done) в”Җв”ҖвҶ’ Teams (notificaciГіn)

Fase 6: VERIFICAR вҖ” Se revisa que todo funciona

Antes de cerrar el proyecto, se prueban las funcionalidades y se revisa la documentaciГіn (CONTEXT.md, etc.).

Fase 7: CERRAR вҖ” Se archiva el proyecto

Cuando todos los hitos estГЎn Done, se marca todo como finalizado y la carpeta del proyecto se mantiene como referencia.
