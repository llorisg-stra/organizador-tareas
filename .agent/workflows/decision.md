# Decisión Técnica

Documenta una decisión técnica de forma rápida.

## Pasos

1. **Preguntar información**:
   - ¿Qué decisión tomaste?
   - ¿Por qué? (razón principal)
   - ¿Qué alternativas consideraste?

2. **Formatear decisión**:
   ```markdown
   ### [FECHA] - [Título corto]
   
   **Decisión**: [Qué se decidió]
   
   **Razón**: [Por qué]
   
   **Alternativas descartadas**:
   - [Alternativa 1]: [Por qué no]
   - [Alternativa 2]: [Por qué no]
   ```

3. **Añadir a CONTEXT.md**:
   - En sección "Notas y Decisiones"
   - Con fecha automática

4. **Actualizar sección técnica** si aplica:
   - Stack, arquitectura, etc.

5. **Confirmar**: "✅ Decisión documentada"

## Notas
- Las decisiones son importantes para el futuro
- Documentar el "por qué" es más importante que el "qué"
